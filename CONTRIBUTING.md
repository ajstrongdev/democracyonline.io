## Pull Requests

We welcome pull requests from the community. However, please note that **we may not always merge submitted PRs**. We carefully review each contribution to ensure it aligns with the project's goals, vision, architecture, and quality standards.

Before submitting a PR, please:

1. Open an issue to discuss your proposed changes
2. Ensure your code follows the existing code style and conventions
3. Test your changes thoroughly
4. Update documentation as needed

## AI-Assisted Contributions

We accept contributions that involve AI assistance (such as code generation tools, AI pair programming, etc.) under the following conditions:

- **Verification**: All AI-generated code must be verified by human sources and thoroughly reviewed
- **Functionality**: The code must work correctly and pass all tests
- **Disclosure**: You must disclose the use of AI tools in your pull request description

We believe AI can be a valuable tool for development, but human oversight and verification remain essential.

## Getting Started

1. Fork the repository
2. Create a new branch for your feature or bugfix
3. Make your changes
4. Submit a pull request with a clear description of your changes

## Questions?

If you have questions about contributing, feel free to open an issue or reach out to the maintainers.

Thank you for helping make this project better!
